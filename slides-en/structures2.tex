\begin{frame}[fragile]\frametitle{Elements}
The non-textual elements, including
\begin{itemize}
\item boxes (\verb|\makebox, \framebox ...|)
\item lines (\verb|\rule |)
\item distances (\verb|\hspace, \hspace*, \vspace, ...|)
\item \ldots
\end{itemize}
are described with commands inside the document.\medskip

Their arguments are given in millimeters, centimeters, pixels, or in proportion
of the font 'M' or 'x' characters,
\begin{center}
10mm, 1.5cm, 12px, 1em, 0.8ex, \ldots
\end{center}
\end{frame}


\begin{frame}[fragile]\frametitle{Examples}
Let us inspect some of the most common commands:

\begin{itemize}
\item \verb|\rule[voffset]{width}{height}| \\ 
  e.g.:  \verb|Baseline text\rule[-5pt]{2cm}{1ex}|: \\
  Baseline text\rule[-5pt]{2cm}{1ex}.\bigskip
\item \verb|\framebox[width][pos]{text}| \\
  e.g.: \verb|\framebox[9cm][s]{foo  bar baz}|:\\
  \framebox[9cm][s]{foo bar baz}\\\medskip
  where s=stretch.The other options are l=left and r=right
\end{itemize}
\end{frame}


\begin{frame}[fragile]\frametitle{Examples\dots}
\begin{itemize}
\item \verb|\makebox[width][pos]{text}| \\
  is the same as \verb|framebox|, but with no borders
\item \verb|\fbox{text}|\\
  is a short form: \verb|\fbox{Inspect this!}| \\
  \fbox{Inspect this!}
\item \verb|\mbox{text}|\\
  The invisible box \verb|\mbox{the contents}| \\ $\mapsto$
  \mbox{the contents} is treated as a single entity. This property is useful,
  and will be needed many times in the future\ldots
\end{itemize}
\end{frame}


\begin{frame}[fragile]\frametitle{Examples\dots}
\begin{itemize}
\item \verb|\vspace*{height} ja \hspace*{width}| \\
  \verb|\hspace*{0.3\textwidth}|\hspace*{0.3\textwidth} adds  
  certain amount of space to the documents\footnote{Observe the relative
  distance measure \texttt{\textbackslash textwidth}}\footnote{Footnotes can be
  produces with \texttt{\textbackslash footnote}\ldots}. Vertical space can be
  added with \verb|\vspace|. {\footnotesize\gray{(Negative distances
  move the following contents to the left or up. The command with star is 
  enforcing, otherwise adding the space
  is optional, and will be omitted at certain places, e.g. at the end of 
  paragraph)}}
\item \verb|\parbox[pos][height][ipos]{width}{text}| \\
  produces a box. For example, \\{\small
  \verb|The \fbox{\parbox[b][1cm][b]{4cm}{Schrödinger's cat}}|
  The \fbox{\parbox[b][1cm][b]{4cm}{Schrödinger's cat}}}
\end{itemize}
\end{frame}


\begin{frame}[fragile]\frametitle{Examples\dots}
\begin{itemize}
\item \verb|pos| can be t=top (line) or b=bottom (line)
\item \verb|ipos| can be t=top, b=bottom, c=centered or
  s=stretched
\item \verb|parbox| behaves like a small embedded
page\footnote{\texttt{\textbackslash minipage} will behave exactly like an
embedded page\ldots}, end it is treated as a single element.
\end{itemize}
\end{frame}

\begin{frame}[fragile]\frametitle{Conclusions}
We saw that \LaTeX{} code becomes complicated, if we
construct detailed elements. 

Because of this reason, the logical content of the document and fiddling with
the details should be kept \textbf{separated}.

In summary,
\begin{itemize}
\item everything can be tuned,
\item it is not wise to tune everything, 
\end{itemize}
\begin{flushleft}
but from experience I suppose that you would like to change the default font
sooner or later\ldots
\end{flushleft}
\end{frame}


\begin{frame}[fragile]\frametitle{Fonts}
The easiest way to get started with different document styles is to load a
pre-defined style package \gray{(from a \texttt{.sty} file)} in the manuscript
preamble:\smallskip
\begin{verbatim}
\usepackage{mystyle}
\end{verbatim}\smallskip
\begin{flushleft}
where the style is possibly one from the list \texttt{helvet, palatino,
avant, charter, bookman, newcent} or \texttt{times} \gray{(that are probably
already installed in your LaTeX-distribution package).}
\medskip

The size of the font can be changed with the descriptive commands
\verb|{\large  }| and \verb|{\small  }|.
\end{flushleft} 
\end{frame}

\begin{frame}[fragile]\frametitle{Fonts\dots}
Observations
\begin{itemize}
\item These commands are declarations, not functions: \verb|\large| = change to
large font and keep in until the end of this (scope or) environment.
\item Anything between the \verb|\begin{}| \dots \verb|\end{}|
-block is an environment. A new environment can also be defined with the curly
braces \{ \dots \}.
\item Use the curly braces and think of the environments. This is better style
(from the programmer's point of view) than scattering the document with commands
like \verb|\normalsize| without a clear reference to the beginning of the
``non-normal'' behaviour.
\end{itemize}
\end{frame}

\begin{frame}[fragile]\frametitle{Fonts\dots}

\begin{frame}[fragile]\frametitle{Conclusions\dots}
A good strategy for writing a paper with \LaTeX{} is to \medskip

\begin{itemize}
\item Make the title page by hand (e.g. by using the command just learned) 
\item Write the text with clean \LaTeX, using perhaps some of the extension
packages from AMS (American Mathematical Society) to stick with portability.
\item Make your own style sheet, if necessary
\item Include minor things in the document preamble before the
  \verb|\begin{document}| clause \gray{(even though it is sometimes legal to
  define command later in the document --- but what is the
  advantage?)}
\end{itemize}
\end{frame}
